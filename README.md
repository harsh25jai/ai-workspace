# ai-workspace

`ai-workspace` is a local Node.js CLI tool that converts any project repository into an AI-ready workspace.

By running standard analysis on your directory, `ai-workspace` rapidly drafts structured markdown components describing your project's architecture, dependencies, rules, and entry points. This prepares any repository to be optimally understood by modern Coding LLM Agents.

## Setup & Installation

Ensure you are running Node 18+:

```bash
# Build the local TS CLI
npm install
npm run build

# Link the CLI globally for ease of use
npm link
```

## CLI Commands

### 1. `ai-workspace init`
Scaffolds out the baseline AI environment inside the `.ai/` directory.

**Generates Placeholder Assets:**
- `.ai/project.md`
- `.ai/architecture.md`
- `.ai/rules.md`
- `.ai/repo-map.json`
- `.ai/skills/`

### 2. `ai-workspace analyze`
The core engine. It parses the current directory recursively while actively ignoring `node_modules` and `.git`. It automatically dictates languages, roots, structured component trees, and dependency footprints, writing the findings securely back into the `.ai/` templates generated by `init`.

### 3. `ai-workspace explain <file>`
Translates individual source files (like `ai-workspace explain src/cli/index.ts`) into a concise, readable graph mapping its active role, dependencies, and imports.
